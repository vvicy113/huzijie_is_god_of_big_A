# 策略与引擎设计

## Strategy 接口

```java
interface Strategy {
    String getName();                        // 策略名称
    StockSelector getStockSelector();         // 选股器
    StockComparator getComparator();          // 打分器
    void evaluate(StrategyContext ctx);       // 评估 → SignalBoard
    default void onReset() {}                 // 回测前重置
}
```

## 接口规范

所有有多个子类的接口必须提供 `getCode()` + `getDesc()`：
- `StockSelector.getCode()` / `getDesc()` — 23个实现
- `StockComparator.getCode()` / `getDesc()` — 6个实现
- `TradeExecutor.getName()` — 可替换执行器

## StrategyContext

```java
class StrategyContext {
    List<String> candidates();                 // 候选列表
    Map<String, List<KLine>> klineMap();       // stockCode → 历史K线
    Map<String, Position> positions();         // stockCode → 持仓
    double availableCash();                    // 可用资金
    LocalDate date();                          // 当前交易日
    SignalBoard signalBoard();                 // 信号板

    // 快捷方法
    KLine todayKLine(String code);
    List<KLine> history(String code);
    Position position(String code);
    boolean isHolding(String code);
    void signal(String code, TradeAction action, double score);
}
```

## TradeSignal / SignalBoard

```java
record TradeSignal(String code, TradeAction action, double score)
    implements Comparable  // 默认按 score 降序

class SignalBoard {
    void add(TradeSignal);               // 添加（同股票保留高分）
    void remove(String stockCode);       // 按代码移除
    TradeSignal first();                 // 最高分
    List<TradeSignal> topN(int n);       // 前 N 名
    List<TradeSignal> all();             // 全部（按分数降序）
    SignalBoard filterBuy(double min);   // 只保留 BUY 且 score>min
    void clear();                        // 清空
}
```

内部：TreeSet 按 score 降序 + HashMap 按 code 快速查找，双索引。

## TradeExecutor

```java
interface TradeExecutor {
    String getName();
    List<TradeRecord> execute(
        SignalBoard board,
        Map<String, Position> positions,
        DoubleRef cash,
        Map<String, Double> prices,
        LocalDate date
    );
}
```

默认实现：`EqualSplitExecutor(maxPositions, cashRatio)`
- 先卖后买，等分资金，100股整数倍
- 支持 replaceLowerScored（满仓卖低换高）
- 所有参数通过 BacktestConfig 配置

## BacktestEngine 每日执行流程

```
1. StockSelector.select(pool, date)              → 候选池
2. 加载每只候选的历史K线（最近60日）               → klineMap + prices
3. 构建 StrategyContext                           → ctx
4. strategy.evaluate(ctx)                         → 填充 SignalBoard
5. applyRiskCheck()                               → 追加风控 SELL
6. executor.execute(board, positions, cash, prices, date)
7. 计算权益（现金 + Σ 持仓×收盘价）
8. 记录权益曲线
回测结束：强制清仓 → MetricsCalculator → ReportGenerator
```

## onReset() 规范

- **无状态策略**（纯函数，无内部计数器）→ 不需要重写
- **有状态策略**（跨日计数器、动态阈值、缓存）→ 必须在 onReset() 中归零
- Engine 在每次回测开始前调用一次
