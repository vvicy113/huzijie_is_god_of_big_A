# 组件调用关系

## 1. 数据导入流程

```
DataMenu.doImport()
  → new CsvImporter()
    → importAll()
      → loadMainBoardStocks()              // 读取 股票列表.csv，筛选 sh/sz 主板
        → 返回 Set<String> allowedCodes
      → 遍历 csv/日k/*.csv
        → importOneFile(file, ps)
          → 读取 GBK/UTF-8 CSV → OpenCSV 解析
          → ps.addBatch() → conn.commit()  // 5000行/批
      → daily_kline 表
```

## 2. 回测流程（设计态）

```
BacktestMenu.show()
  → 用户输入日期范围
  → StrategyRegistry 提供策略列表
  → Strategy.getStockSelector()              // 获取选股器

BacktestEngine.run(strategy, from, to)
  ┌─ for each trading day ──────────────────────────┐
  │                                                    │
  │ 1. StockSelector.select(candidates, date)          │
  │    → 从 daily_kline 批量 SQL 过滤                  │
  │    → 返回候选股票列表                              │
  │                                                    │
  │ 2. CsvDataLoader.loadDailyKLine(code, from, date)  │
  │    → KLineRepository.findByStockCode()             │
  │    → 为每只候选加载历史 K 线                        │
  │                                                    │
  │ 3. new StrategyContext(candidates, klineMap, ...)  │
  │                                                    │
  │ 4. Strategy.evaluate(ctx)                          │
  │    → 往 SignalBoard 填入 TradeSignal               │
  │                                                    │
  │ 5. SignalBoard → TreeSet<TradeSignal> (按score降序)│
  │    → Engine 从高到低读取                            │
  │    → BUY: 资金够→买入，不够→跳过                    │
  │    → SELL: 卖出持仓                                 │
  │                                                    │
  │ 6. 记录当日权益曲线                                  │
  └────────────────────────────────────────────────────┘
  
  → MetricsCalculator.calculate()
  → ReportGenerator.generate()
  → 输出回测报告
```

## 3. 关联分析流程

```
AnalysisMenu.show()
  → 用户输入主股票 + 对比股票列表
  → RelationAnalyzer.analyze(primaryCode, compareCodes)
    → EastMoneyFetcher.fetchBasicInfo()      // 股票基本信息
    → EastMoneyFetcher.fetchIndustry()       // 行业信息
    → EastMoneyFetcher.fetchConcepts()       // 概念板块
    → EastMoneyFetcher.fetchIntradayData()   // 分时数据
    → 行业匹配 / 概念重叠检测
    → CorrelationCalculator.calculate()      // Pearson 相关性
    → LeadLagDetector.detect()              // 领涨跟风
    → LlmReportService.analyze()            // AI 综合分析
  → 输出 StockRelationReport
```

## 4. 选股器链执行流程

```
DefaultSelectorRegistry reg = new DefaultSelectorRegistry("短线");
reg.register(ALL_MAIN_BOARD);    // code=1
reg.register(VOL_20_1P5);        // code=201
reg.register(MA_5_20);           // code=401

StockSelector chain = reg.build();
  → new StockSelectorChain(-1, "短线", [
        AllMainBoardSelector,
        VolumeThresholdSelector(20,1.5),
        MaAlignmentSelector(5,20)
    ])

chain.select([], date)
  ├─ AllMainBoardSelector.select([], date)
  │    → SELECT DISTINCT stock_code FROM daily_kline
  │    → 3000 只
  ├─ VolumeThresholdSelector.select([3000只], date)
  │    → WHERE volume >= AVG(20d) * 1.5
  │    → 500 只
  └─ MaAlignmentSelector.select([500只], date)
       → WHERE short_ma > long_ma
       → 200 只
```

## 6. 数据存储结构

```
data/stocks.db (SQLite)
  └── daily_kline
        (stock_code, date, open, high, low, close, volume, amount)
        PRIMARY KEY (stock_code, date) WITHOUT ROWID
        INDEX idx_kline_stock (stock_code)

csv/日k/*.csv          ← 数据源 (UTF-8, 5836个日K线文件)
csv/股票列表.csv        ← 导入过滤依据
csv/周k/  csv/月k/     ← 暂未启用
```
